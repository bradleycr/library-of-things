# Contributing

Thanks for considering contributing to Flybrary.

## How to contribute

1. **Open an issue** (or comment on an existing one) so we’re aligned on the change or bug.
2. **Fork the repo**, create a branch off `main`, and make your changes.
3. **Run `pnpm build`** before pushing to ensure the app builds.
4. **Open a PR** against `main`. Use the PR template; a short description of what and why is enough.

## Reporting bugs

Open an issue with a clear title and steps to reproduce (and your environment if relevant). If the app is deployed, note whether the bug is on the live site or only local.

## Suggesting features

Open an issue with a short description of the feature and why it would help. Discussion there before a PR is welcome.

## Code style

No formal style guide. Match the existing code (indentation, naming, comments) and keep it simple.

## PR process

- Target branch: `main`.
- CI may run on your PR; ensure the build passes.
- A maintainer will review and merge when appropriate.
