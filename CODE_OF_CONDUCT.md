# Code Of Conduct

Library of Things is for people building trust-based physical sharing systems.
The project is small, practical, and community-oriented; collaboration should
feel the same.

## Expected Behavior

- Be kind, direct, and generous with context.
- Assume good intent, especially from new contributors.
- Keep discussions focused on the project and the people using it.
- Prefer clear tradeoffs over winning arguments.
- Respect privacy. Do not post private contact details, secrets, or real member
  data in issues, pull requests, logs, or screenshots.

## Unacceptable Behavior

- Harassment, threats, discrimination, or personal attacks.
- Publishing another person's private information.
- Repeatedly derailing issues or pull requests after maintainers ask to refocus.
- Pressuring communities toward surveillance, punitive lending, or other
  behavior that conflicts with the project's trust-based design.

## Reporting

If something feels unsafe or off-topic, open a private channel with a maintainer
if you have one, or open a GitHub issue with only the information you are
comfortable sharing publicly.

Maintainers may remove comments, close issues, or block participants to keep the
project welcoming and useful.
